# GlowUp AI — 产品需求文档 (PRD v2)

> **一句话定位：拍张自拍，上传你喜欢的妆，AI 帮你画在自己脸上。**

---

## 1. 项目概述

### 1.1 项目名称
**GlowUp AI — Your Personal Makeup Artist**

### 1.2 产品定位
GlowUp AI 是一款以 **"灵感复刻"为核心主线** 的 AI 美妆顾问产品。

用户上传一张喜欢的妆容灵感图 + 自己的自拍，AI 自动完成：
1. 分析灵感妆容的风格要点
2. 适配到用户的面部特征
3. 输出可执行的步骤教程
4. 直接在用户脸上生成试妆预览
5. 推荐所需产品，形成购物清单

其他功能（面部分析、明星匹配、发型试戴、库存管理）围绕主线展开，作为"深入探索"和"辅助决策"模块存在。

### 1.3 核心价值主张

> 不是"生成一张好看的图"，而是——
> **让用户知道什么适合自己、为什么适合、怎么做出来，并在购买前看到效果。**

## 2. 背景与问题定义

### 2.1 用户面临的核心问题

| # | 问题 | 痛点等级 |
|---|------|---------|
| 1 | 看到喜欢的妆容，不知道自己画出来什么效果 | 🔴 核心痛点 |
| 2 | 不知道什么风格适合自己的脸型/五官/肤色 | 🔴 核心痛点 |
| 3 | 试错成本高（染发、买错色号、模仿失败） | 🟡 高频痛点 |
| 4 | 化妆品囤积混乱，重复购买 | 🟡 高频痛点 |
| 5 | 教程泛化，缺乏个性化 | 🟠 中频痛点 |

### 2.2 现有解决方案的不足

- **美妆教程（小红书/YouTube）**：泛化、不适配个人特征
- **滤镜 App（FaceApp/SNOW）**：只改外观，不告诉你怎么做
- **专业化妆师**：成本高、不可持续
- **电商推荐**：基于消费数据，不基于面部特征

### 2.3 GlowUp AI 的差异化

> **从"看到灵感"到"画在我脸上"到"告诉我买什么"，一条线走完。**

## 3. 目标用户

### 3.1 主要用户画像

#### 用户 A：灵感驱动型（核心用户）
- 经常刷到好看的妆容、明星造型、博主 look
- 想复刻但不知道适不适合自己
- 核心诉求：**"这个妆画在我脸上是什么样？"**

#### 用户 B：美妆新手
- 想开始化妆但不知道从哪里开始
- 需要结构化引导
- 核心诉求：**"告诉我第一步该干什么"**

#### 用户 C：囤货整理型
- 拥有大量化妆品
- 想管理库存、避免浪费
- 核心诉求：**"帮我理清我有什么、还缺什么"**

### 3.2 优先级
**A > B > C**（产品设计和体验优先满足 A）

## 6. 功能需求

### 6.1 主线：Inspiration Lab

#### 6.1.1 概述
将"参考妆容图"转化为"适合用户自己的可执行妆容方案"。
这是产品的核心体验，必须做到流畅、惊艳、可信。

#### 6.1.2 用户流程

```
Step 1: 上传灵感图（可选）+ 文字描述（可选）
        → 至少提供一项
        → 无灵感图时引导进入 Quiz fallback

Step 2: 上传自拍 或 使用已有 Profile Photo
        → 自动验证人脸有效性
        → 验证失败时给出明确指引

Step 3: AI 处理
        → 分析灵感图风格要点
        → 分析用户面部特征
        → 将风格适配到用户特征
        → 生成结构化输出

Step 4: 结果页
        → 风格名称 & 解读
        → 色盘
        → 分步骤教程
        → 推荐产品列表
        → [Virtual Try-On] 按钮
        → [Add to Wishlist] 按钮

Step 5: 试妆
        → 点击后生成带妆效果图
        → 用户看到"这个风格在我脸上的样子"

Step 6: 分支探索
        → Face Analysis / Hair Lab / Celebrity 入口
        → 管理库存入口
```

#### 6.1.3 输入

| 输入项 | 必需 | 说明 |
|--------|------|------|
| 灵感图 | 二选一 | JPEG/PNG |
| 文字描述 | 二选一 | 如 "Golden hour glow with smoked-out liner" |
| 用户自拍 | ✅ 必需 | 需通过人脸验证 |

#### 6.1.4 输出

| 输出项 | 说明 |
|--------|------|
| 风格名称 | 如 "Soft Glam Sunset" |
| 风格描述 | 2-3 句话概括 |
| 色盘 | 3-5 个核心色 |
| 步骤教程 | 5-8 步，每步含具体操作 |
| 推荐产品 | 品牌 + 产品名 + 品类 |
| 试妆图 | AI 生成的带妆效果 |

#### 6.1.5 成功标准
- 流程完成率 > 40%
- 试妆触发率 > 60%
- 用户主观评价："这看起来确实适合我" > 70%

#### 6.1.6 风险与应对

| 风险 | 应对 |
|------|------|
| 灵感图风格模糊 | 文字描述辅助 + AI 主动追问 |
| 自拍质量差 | 验证环节 + 明确拍摄指引 |
| 试妆图不像本人 | 降低预期措辞 + 持续优化模型 prompt |
| AI 输出空泛 | 结构化 prompt + 强制要求具体产品和步骤 |

---

### 6.2 Quiz（风格发现 / Fallback）

#### 概述
当用户没有灵感图时的替代路径。通过问卷收集偏好，生成 Beauty Blueprint。

#### 触发条件
- 用户在 Landing 点击"不知道想要什么？先做个测试"
- 或在 Inspo Lab 中未上传灵感图时自动引导

#### 输入
- 肤色倾向
- 眼型 / 脸型
- 风格偏好（clean girl / glam / editorial / soft / bold）
- 场景（日常 / 约会 / 通勤 / 晚宴 / 派对）

#### 输出
- Beauty Blueprint（风格名 + 描述 + 色盘 + 教程 + 产品推荐）
- 与 Inspo Lab 结果页共用同一结果组件

#### 成功标准
- 问卷完成率 > 60%
- 完成后进入试妆的比例 > 30%

---

### 6.3 Face Analysis（面部深度分析）

#### 概述
基于自拍的面部几何与色彩理论分析。

#### 触发位置
- 主线结果页底部："想更深入了解你的面部特征？"
- 探索区独立入口

#### 核心能力
- 脸型识别
- 眉形分析
- 肤色冷暖调 / 季型分析
- 适配推荐：首饰颜色、服装色系、眼镜款式、发色、眉形优化

#### 输出
结构化面部分析报告页

#### 注意事项
- 涉及外貌判断，措辞必须正向、建设性
- 避免"你的脸型不好"类表达，改为"你的脸型最适合 XX"

---

### 6.4 Celebrity Twin Scan（明星风格匹配）

#### 概述
找到与用户最相似的明星，并给出基于该明星风格的妆容建议。

#### 触发位置
- 主线结果页侧边："顺便看看你像哪个明星"
- 探索区独立入口

#### 输入
用户自拍

#### 输出
- 明星匹配结果
- 明星经典妆容风格解析
- 针对用户的适配版建议

#### 价值
- 娱乐性强，社交传播潜力大
- 适合作为"惊喜时刻"而非主线

---

### 6.5 Hair Lab（虚拟发型/发色）

#### 概述
在用户自拍上尝试不同发型与发色组合。

#### 触发位置
- 结果页延伸："要不要顺便试试换个发色？"
- 探索区独立入口

#### 输入
- 用户自拍
- 目标发型
- 目标发色

#### 输出
改造后的发型预览图

#### 成功标准
- 单次会话平均尝试 ≥ 2 种组合
- 结果保存/分享率 > 20%

---

### 6.6 Smart Vanity（库存管理）

#### 概述
帮用户管理已有化妆品，建立"数字化梳妆台"。

#### 触发位置
- 推荐产品时："你可能已经有类似的产品？扫一下你的化妆台"
- 探索区独立入口

#### 输入方式
1. 拍照扫描（AI 识别品牌/产品名/品类）
2. 手动录入（AI 标准化产品信息）

#### 输出
- 库存清单
- 品类标签
- 后续推荐中优先调用已有产品

#### 未来延展
- 保质期提醒
- 使用频率统计
- 平替推荐
- 缺失品类检测

---

### 6.7 Wishlist（购物清单）

#### 概述
保存 AI 推荐的产品，形成后续购买决策清单。

#### 触发位置
- 结果页中每个推荐产品旁的 [+] 按钮
- 不作为独立入口，嵌入主流程

#### 输出
- 产品名 / 品牌 / 品类 列表
- 后续可对接电商链接

---

### 6.8 Virtual Try-On（虚拟试妆）

#### 概述
根据 AI 推荐方案，在用户自拍上生成带妆效果图。

#### 触发位置
- 结果页中的 [Try On] 按钮
- 不作为独立入口

#### 输入
- 用户自拍
- 妆容风格描述

#### 输出
带妆后的用户图像

#### 风险
- 图像生成稳定性
- 五官还原度
- 管理用户预期（非照片级精修）

## 8. 统一自拍管理

### 8.1 问题
当前每个模块独立要求上传自拍，体验割裂。

### 8.2 方案
- 引入 **Master Profile Photo** 概念
- 用户首次上传自拍后，自动存为 Profile Photo
- 后续所有模块默认复用，支持临时替换
- Header 中显示 Profile Photo 缩略图，点击可更换

### 8.3 规则
- 所有需要自拍的流程：优先使用 Profile Photo
- 用户可在任意流程中"使用其他照片"（临时）
- 临时照片不覆盖 Profile Photo，除非用户主动设置

## 10. 非功能需求

### 10.1 性能
- 主线全流程（上传 → 结果）目标 < 15s
- 试妆生成目标 < 10s
- 加载中提供进度条 + 文案提示，降低等待焦虑

### 10.2 易用性
- 新用户 **60 秒内** 完成首次核心体验
- 自拍验证失败时给出 **具体改进指引**（不只是"请重新上传"）
- 所有 AI 输出结构化、可执行，避免空泛赞美

### 10.3 稳定性
- 单模块失败不影响其他模块
- AI 超时/失败时提供明确 fallback 和重试选项
- 网络异常时本地数据（库存、Wishlist）不丢失

### 10.4 隐私与安全
- 用户自拍属于 **高敏感数据**
- 需明确告知：上传用途、是否存储、如何删除
- 建议：分析完成后不在服务端保留原始图像
- 本地优先：库存和 Wishlist 数据优先存本地

## 11. 用户体验原则

### 11.1 三个"第一次"
- **第一眼要"美"**：首页即高级感
- **第一次分析要"准"**：输出具体、可信、不空泛
- **第一次输出要"能用"**：教程可执行、产品可购买

### 11.2 措辞原则
- 永远正向（"你最适合 XX" 而非 "你不适合 YY"）
- 具体胜于笼统（"用 #B5838D 色系腮红打在颧骨偏上位置" 而非 "用适合你的腮红"）
- 诚实胜于讨好（如果灵感妆确实不太适合，说明原因并给替代建议）

## 12. 风险与应对

| 风险 | 等级 | 应对策略 |
|------|------|---------|
| AI 结果可信度不足 | 🔴 高 | 结构化 prompt + 强制输出格式 + 用户反馈机制 |
| 自拍质量不稳定 | 🟡 中 | 前置验证 + 拍摄指引 + 示例图 |
| 试妆图不像本人 | 🟡 中 | 管理预期（"风格预览"而非"精确效果"）+ 迭代模型 |
| 用户把产品当功能菜单 | 🟡 中 | 首页聚焦设计 + 引导文案 |
| 隐私担忧影响上传意愿 | 🟡 中 | 隐私说明前置 + 本地处理优先 |
| 商业闭环缺失 | 🟠 低（当前阶段） | V2 再解决，V1 先验证核心体验 |
